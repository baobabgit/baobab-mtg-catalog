# Checklist interne — readiness audit v1.0.0 (GO / NO GO)

Document de travail pour valider qu’une release **1.0.0** peut être publiée sans surprise. Cocher au fil des itérations ; une case non satisfaite peut justifier un **NO GO** ou une action correctrice avant tag.

## Périmètre produit

- [ ] Le README décrit clairement périmètre inclus / exclu et le rôle dans l’écosystème Baobab.
- [ ] La distinction `CardDefinition` / `CardPrinting` / `Set` est documentée et reflétée dans le code.
- [ ] L’import idempotent et la consultation (façade + filtres) sont illustrés par des exemples exécutables ou testés.

## Qualité et conformité

- [ ] `pytest` passe avec couverture **≥ 90 %** (seuil `pyproject.toml`).
- [ ] `black`, `flake8`, `mypy` (`src`), `pylint` (`src` + `tests`), `bandit` (`src`) : aucune erreur bloquante.
- [ ] Contraintes `docs/000_dev_constraints.md` respectées (structure, une classe par fichier, types, docstrings publiques, exceptions racine).

## API publique et stabilité

- [ ] Les exports documentés du package racine (`__all__`) sont stables et intentionnels.
- [ ] Pas de fuite obligatoire d’implémentations in-memory pour un usage production documenté (injection de repositories explicitée).
- [ ] `CHANGELOG.md` à jour pour la version publiée ; version `pyproject.toml` alignée.

## Dépendances et sécurité

- [ ] Plages de versions des dépendances runtime (`baobab-scryfall-api-caller`, Python) revues.
- [ ] Aucune dépendance interdite par le périmètre (UI, HTTP métier, collection, moteur de règles) n’a été introduite.

## Documentation et support développeur

- [ ] `docs/001_specifications.md` cohérent avec le comportement livré.
- [ ] Journal `docs/dev_diary.md` mentionne les décisions d’architecture récentes.
- [ ] Procédure d’installation et de tests reproduite sur environnement propre (CI ou machine vierge).

## Release technique

- [ ] Tag Git `v1.0.0` prévu sur le bon commit ; build wheel/sdist OK.
- [ ] Publication registre (PyPI interne ou public selon politique Baobab) validée en dry-run si applicable.
